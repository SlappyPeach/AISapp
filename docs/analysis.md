# Product Summary

## Назначение

Текущая версия `AISapp` представляет собой продуктовую сборку АИС для:

- учета материалов;
- документооборота по закупке и поставке;
- складских операций;
- списания по нормам;
- учета спецодежды;
- архива и отчетности.

## Архитектурный итог

Приложение собрано на:

- `Django 6`
- `Django REST Framework`
- `PostgreSQL`
- `Celery`
- `Redis`
- `python-docx`
- `XlsxWriter`

Основной код расположен в `apps/core/`, конфигурация — в `config/`.

## Что входит в текущую продуктовую версию

- ролевая авторизация;
- справочники;
- договоры СМР и договоры поставки;
- заявки на закупку;
- кабинет поставщика и документы поставщика;
- первичные документы;
- приход и отпуск материалов;
- списание по нормам;
- учет спецодежды;
- журнал работ;
- архив и workflow статусов;
- отчеты и экспорт;
- резервное копирование и восстановление;
- REST API для ключевых сущностей.

## Product-hardening, выполненный перед сдачей

В проекте дополнительно приведены в порядок:

- ролевой access layer;
- отчетный слой;
- server-side autosave;
- валидация связанных документов;
- защита от отрицательных остатков;
- поддержка `.env`;
- документация и структура поставки.

## Артефакты, оставленные в проекте осознанно

- `legacy/` — архив старой реализации;
- `.env.example` — шаблон конфигурации;
- `docker-compose.yml` и `Dockerfile` — для контейнерного стенда.

## Рекомендация по сопровождению

Для дальнейшего развития без изменения текущего продуктового контура логично выносить следующими этапами:

- структурированный табличный ввод позиций вместо текстового `textarea`;
- расширение e2e-покрытия интерфейса;
- отдельный production-runner поверх WSGI/ASGI и reverse proxy.
